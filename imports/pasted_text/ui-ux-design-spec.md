Para diseñar una maqueta (UI/UX) exhaustiva y precisa de este sistema, es fundamental dividir las vistas según los tres roles principales descritos en el documento: Supervisor (aplicación móvil/PWA), Recursos Humanos (aplicación web de escritorio) y Administrador (aplicación web de escritorio). El Empleado/Operario no requiere vistas en pantalla, ya que su única interacción es física mediante el reconocimiento facial en el dispositivo FaceDeep.Considerando que la interfaz web utilizará React con la librería de componentes profesionales Mantine, las maquetas deben proyectar un diseño limpio, modular y enfocado en la usabilidad de datos masivos.A continuación, detallo extensamente cada pantalla que debes maquetar:1. Vistas Generales y de AutenticaciónPantalla de Inicio de Sesión (Login)Diseño general: Diseño centrado, responsivo tanto para escritorio como para móvil.Componentes: Campos de texto para Usuario y Contraseña, botón primario de "Iniciar Sesión" y enlace funcional para "Cambio de contraseña".Lógica visual: Al autenticarse, el sistema debe redirigir a una navegación adaptada exclusivamente a las capacidades de cada rol.2. Vistas para el Supervisor (App Mobile / PWA)Esta aplicación debe diseñarse con una mentalidad "Mobile-First", ya que será una Progressive Web App instalada en el celular del supervisor. Debe contemplar indicadores visuales para el funcionamiento sin conexión.Pantalla Home / Apertura de JornadaCabecera: Nombre de la estación asignada y estado de conexión a la red.Sección Central: Botón de acción principal grande para "Abrir Jornada / Habilitar sistema".Lista Sugerida: Un listado en formato grilla o tarjetas mostrando la cuadrilla de operarios sugerida basada en el historial de esa estación.Interacciones: Casillas de verificación (checkboxes) tipo "switch" junto a cada operario para agregarlos o quitarlos de la cuadrilla del día antes de abrir la jornada.Panel de Jornada en Vivo (Centro de Control Móvil)Cabecera: Temporizador de la jornada activa e indicador de estado del dispositivo físico FaceDeep (Disponible/Bloqueo Antiduplicado).Lista de Marcaciones: Tarjetas que se actualizan en vivo conforme los operarios fichan.Sistema de Colores Estricto: Cada tarjeta de operario debe tener un borde o fondo semitransparente basado en reglas: Verde (marca normal sin irregularidades), Amarillo (contiene una observación por tardanza o salida anticipada) o Rojo (ausencia no justificada).Botón Flotante (FAB): Opción siempre visible para "Incorporación Inesperada", útil para añadir operarios de reemplazo de otras cuadrillas.Pantalla de Observaciones (Modal o Pantalla Completa)Contexto: Se dispara obligatoriamente ante tardanzas mayores a 15 minutos, salidas anticipadas o doble turno.Atajos Rápidos: Botones tipo píldora (chips) con motivos frecuentes cargados previamente (ej. "Cuadrilla completa terminó a las 12:00", "Cambio de turno") para agilizar la carga en la calle.Entrada de Datos: Un área de texto libre (no tabulada) amplia, optimizada para el teclado táctil del celular, obligatoria para registrar el motivo exacto de la irregularidad.3. Vistas para Recursos Humanos (Web Desktop)Esta interfaz debe priorizar la visualización de alto volumen de datos, las tablas densas y los indicadores de rendimiento.Dashboard Principal (Inicio RRHH)Tarjetas de KPIs (Arriba): Bloques visuales mostrando métricas generadas a partir de las observaciones, como porcentajes de asistencia, tardanzas sistemáticas, horas trabajadas y patrones de ausencia.Tabla Central de Registros Prioritarios: Una tabla de datos ancha. Los registros que poseen observaciones pendientes de revisión deben estar obligatoriamente destacados visualmente (ej. fondo sutil amarillo o un ícono de alerta rojo).Registros Normales: En esta misma vista o en una pestaña adyacente, los registros normales (flujo sin observaciones) deben mostrarse en segundo plano visual.Pantalla de Carga de JustificacionesFiltros de Selección: Selectores para buscar al empleado y establecer el rango de fechas del período.Selector de Tipos: Un menú desplegable para asignar justificaciones precargadas (vacaciones, enfermedad, licencia, accidente de trabajo/RT, salida gremial).Acción: Al presionar "Guardar", debe existir un feedback visual indicando que los datos ya están disponibles en las vistas expuestas para el sistema DAS.Ficha de Legajo (Vista Detalle del Empleado)Cabecera de Perfil: Foto, Nombre, Número de Legajo/DNI, Categoría (ej. peón, chofer) y Estado actual, traídos en tiempo real desde el sistema DAS.Resumen Mensual: Gráficos pequeños o indicadores numéricos de su asistencia en el mes.Historial de Actividad (Pestañas): Pestañas para navegar entre marcaciones históricas, ausencias justificadas y una sección directa a todas las observaciones libres creadas por los supervisores.Línea de Tiempo: Elemento visual que muestra los eventos cronológicos del empleado.4. Vistas para el Administrador (Web Desktop)El administrador tiene control total y gestiona el hardware, la infraestructura lógica y los accesos.Pantalla de Mapa Interactivo de EstacionesContenedor Principal: Un mapa geográfico interactivo de la ciudad.Marcadores (Pins): Puntos de ubicación para cada estación de barrido.Semáforo de Estado: Los marcadores deben tener colores que indiquen el estado en tiempo real de los dispositivos FaceDeep (online / offline).Panel Lateral: Formulario deslizable para alta o edición de estaciones, que se rellena automáticamente al hacer clic en el mapa (selector en mapa).Panel de Control de Dispositivos FaceDeepListado de Hardware: Tabla técnica mostrando todos los dispositivos conectados.Detalles por Fila: Columnas que indiquen la capacidad de almacenamiento biométrico restante, versión de firmware y estado de sincronización.Acciones Rápidas: Botones en cada fila para forzar sincronizaciones, reiniciar o aplicar acciones remotas.Flujo Visual de Enrolamiento Facial (Alta de Operarios)Paso 1 - Verificación DAS: Campo para ingresar Legajo/DNI. Botón para consultar al DAS y traer datos vivos. Si el estado es "Baja", mostrar alerta bloqueante.Paso 2 - Captura Física: Interfaz que se comunique con el "FaceDeep Padre". Visualizador de confirmación cuando el dispositivo captura exitosamente el Rostro y el DNI.Paso 3 - Completar Perfil: Formulario con los datos obligatorios (Nombre, Categoría, Función) y botón "Guardar Perfil Completo" para iniciar la sincronización masiva a todos los dispositivos.Visor de Auditoría y ConfiguraciónPantalla de Auditoría: Tabla de logs compleja con filtros. Columnas obligatorias: Usuario, Acción realizada, Fecha/Hora, y botón para desplegar el "detalle del cambio" asegurando la trazabilidad de cada edición.Configuración Global: Formularios simples para definir el umbral de minutos exactos que disparan las "tardanzas sistemáticas" y parámetros generales de los dispositivos.
